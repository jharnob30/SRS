  1. Introduction
1.1 Purpose

Due to Covid19 we cannot go to university for classes. So, we need to do our academic work online. The purpose our app is to take quiz and other exams in a more untroubled way, to lessen the difficulties in online and to ensure the fairness of the exam and to track the grades and questions of exams.

1.3 Intended Audience and Reading Suggestions

Our targeted consumers are educational institute or other groups who teaches and want to evaluate their teachings. Simple English and knowledge of UML diagram is required to understand srs.

1.4 Product Scope

Purpose- The main purpose of the app is to make fairness of exams and taking attendance using face recognition and checking plagiarism with google reverse check.

Business strategy- Every educational institutes can use our app to take online exams with serenity. In future we can upgrade this app and there’ll be premium version(paid) with new features.

Corporate goals- Faculty don’t have to take attendance manually and he/she doesn’t need to check plagiarisms.

  1. Overall Description
2.1 Product Perspective
  • Our app is new self-contend product.
  • The app is to provide better environment for taking exams online.
  • Student attendance is automated so faculty doesn’t have to take manually
  • Our app is a standalone system

 

2.2 Product Functions
  • Automated attendance using camera
  • Plagiarism checker
  • Auto closing session if student leaves the tab
  • Uploaded document auto converts into image file.

 

2.3 User Classes and Characteristics

There will be two type of user in our app-

  • Teacher
  • Student

 

Signup- To register an account to access the app.

Login- To sign into the app and use it

Logout- To sign-out from the app

CreateQuiz- To create questions (Teacher)

UploadQuiz- To upload questions that will appear in front of students in the app(teacher)

DraftQuiz- To save incomplete questions(teacher)

GradeQuiz- To grade the answers that student submitted.

SelectSections- To select specific course section

ArchiveQuiz- Where quiz history will be saved

StartQuiz- To start giving exam(student)

SubmitQuiz- To submit answers(Student)

ViewResults- To see grades (student)

 

 

2.4 Operating Environment

Our software is web-based. It will run on a x32 or x64 bit architecture microprocessor, minimum ram needed is 4gb. It will acquire the computers audio and video components to access information. Our Android app will run from OS Lollipop version.

2.5 Design and Implementation Constraints

Our developers will be maintaining the delivered software. Developers cannot use platform independent language for developing the software.

2.6 User Documentation

A online help page will be provided along with the software when it will be delivered for the user to operate the software.

2.7 Assumptions and Dependencies

Our software will 3rd party components for attendance taking, converting the teacher uploaded (questions) files to image, plagiarism check via google.

  1. External Interface Requirements
3.1 User Interfaces

The user interface is easy to understand and use. A manual will be provided for the user. Here are some mock-up GUI designs for the software -Mockup Designs

3.2 Hardware Interfaces

Our product is web software it will run on every computer OS. Software will use camera, audio and user’s information for log in and out and registration. The mobile version of our software will only be available Android Lollipop version OS. It will use the phones camera, audio, OS features to run properly.

3.3 Software Interfaces

Django frame-work's libraries will be used to make the data flow between the database and UI. Some information of the user will pre-exist in our database. User logs in and out in the software data will communicate between the UI and database. 3rd party API'S will be then used for required actions in the software and will pass the data for validation and submission. If the requirements are matched user will be shown the next UI. Our Android version will work in a similar way. We will use the Katlin’s libraries for data manipulation. Data's will be shared with 3rd party components.

3.4 Communications Interfaces

The software(system) will use e-mail for communicating with users for notifying and quires. FTP will be used in our software for uploading files.

  1. System Features

Our software's features explained by the combination of use case, UML and feature hierarchy. Attendance will be automated; teachers will take quiz and the answer will be checked for plagiarism. It will have a separate class of its. Fairness of the exam will be handled with session handling and questions in image form.

 

4.1 System Feature "Attendance"

Using face recognition, we will take attendance of the students. We can make sure that the student is taking the exam and no one else.

4.1.1      Description and Priority

A 3rd party component will be used to take the attendance of the students. They have open their camera to give attendance this will ensure that the same student who signed up for the course is taking the exam. If it’s not the same student then a e-mail will be sent to the teacher about this. This feature will help the teacher not to take the attendance manually (Rate 9).

4.1.2      Stimulus/Response Sequences

Students will click the quiz link and will the redirected to a quiz taking page. Where they click on their right answer and submit it for grading to the teacher.

 

4.1.3      Functional Requirements      

REQ-1: Students have to have camera for attendance. If the same students is not present then a e-mail will be sent to the teacher, or if there are some technical errors it will redirect to the page again.

4.2 System Feature "plagiarism"

Students written answers will be checked in google for maximum similar copy.

 

4.2.1      Description and Priority

It’s to make sure the fairness in the exam. Teacher can evaluate how much the students are understanding.

4.2.2      Stimulus/Response Sequences

After the paper is submitted to system will check it in google and report to the teacher.

 

4.2.3      Functional Requirements      

REQ-1: Written quiz has to be taken by the students to check for plagiarism. If plagiarism is found teacher will be shown that.

4.3 System Feature " Stopping SS"

To stop taking ss a session will start. if a student changes from quiz tab to other tab the quiz session will stop and teacher will get e-mail.

 

4.3.1      Description and Priority

Exam fairness is main priority student must stay in the quiz tab while taking the quiz. if he changes tab, he will take to login page and teacher will get e-mail.

4.3.2      Stimulus/Response Sequences

If a student changes the quiz tab. He/She will be taken to login page.

 

4.3.3      Functional Requirements      

REQ-1: If the quiz tab is changed. Student will be redirected to login page.

4.4 System Feature " Stopping crtl+v(copy paste)"

Teachers uploaded docx or pdf file will be converted to image file by a 3rd party component. Its to have fairness in the exam.

 

4.4.1      Description and Priority

Students will no longer be able to copy the text and check it in google for answers.

4.4.2      Stimulus/Response Sequences

Teacher upload the file for quiz it will be shown to the students as an image file.

 

4.4.3      Functional Requirements      

When the teacher uploads a docx or pdf file it will be converted into an image file.

 

 

 

  1. Other Nonfunctional Requirements
5.1 Performance Requirements

For a better performance the user must have an uninterrupted and good internet connection. To use this app web cam or a camera source is required. A clear and better resolution camera will work well for face recognition. Students have to type their answers in the quiz form and just press the option ‘Submit’ to submit the answer therefore there is no hassle of uploading any image or answer file.

5.2 Safety Requirements

A problem that can be faced by the students is that if there is a sudden internet or electricity failure

Then the student would not be able to take the quiz as the quiz window would be closed which is a loss for the students and can hamper their grades.

 

To be determine later for more safety issues if found.

 

5.3 Security Requirements

There is login authentication. The user name, email and password would be unique for every user.

If any of the information doesn’t match while a user is trying to login the system would notify the user by saying ‘Your password is incorrect’ or ‘Your user name is incorrect’.
